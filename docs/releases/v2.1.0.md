# Release Notes — v2.1.0

**Date:** 2026-04-07

## Summary

v2.1.0 adds two new game store sources (GOG Galaxy and Xbox PC Game Pass), a browser-based configuration page served by the sidecar, and several UX improvements to the store filter panel.

## New features

### GOG Galaxy source

New `--gog-refresh-token` flag (or `[gog] refresh_token` in `config.toml`) fetches your GOG library via the GOG OAuth2 + embed API. Games are tagged `gog` and enriched with a Steam AppID via the existing resolver chain. The setup wizard (step 3/6) guides through the one-time browser auth flow. A dedicated `🌌 GOG` filter button appears in the library and alerts panels.

### Xbox PC Game Pass source

New `--game-pass` flag (or `gamepass = true` in `config.toml`) fetches all current PC Game Pass titles from the public Microsoft catalog. No authentication required. Games are tagged `gamepass`. A dedicated `🎯 Game Pass` filter button appears in the filter panels.

### Web configuration page (`/config`)

A browser form served by `steam-serve` at `/config` lets users view and edit all credentials and settings without touching the TOML file directly. Accessible without authentication in bootstrap mode (no `serve_token` configured). Credential fields are shown as `● ● ●` placeholders when already set and are never pre-filled. Saved via `POST /api/config`; masked values are never written back to disk.

### Fetch-progress bandeau

The library page now shows a fixed top banner while a background fetch is running. The banner displays current progress (`idx/total` + game name) and automatically reloads the page when the fetch completes. Powered by a new public `GET /api/status` endpoint.

### Store filter multiselect toggle

A small "multi" checkbox next to the Store filter label switches between exclusive mode (click = select only that store, default) and multi-select mode (click = toggle individual store independently). State is persisted in `localStorage` and the URL hash.

## Fixes

- **Bool coercion in `/api/config`** — strict parsing: only `"true"`/`"1"` → `True`, `"false"`/`"0"` → `False`; any other string returns HTTP 400.
- **Store multiselect exclusive mode** — unchecking the multiselect toggle now correctly collapses to a single active store button instead of leaving multiple active.
- **GOG OAuth uses POST** — token exchange and refresh requests now correctly use `POST` with a form body instead of `GET` with query parameters.
- **GOG client credentials overridable via env** — `STEAMPULSE_GOG_CLIENT_ID` and `STEAMPULSE_GOG_CLIENT_SECRET` environment variables override the built-in defaults.
